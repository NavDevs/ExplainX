1. Architecture Overview
ExplainX follows the standard Chrome Extension MV3 architecture with four distinct layers that communicate via the Chrome Messaging API.
1.1 Component Map
Component
File
Responsibility
Service Worker
background.ts
Registers context menu; listens for menu click events; relays messages to content scripts; makes AI API calls.
Content Script
content.ts
Injected into all pages; captures selected text; renders the explanation popup overlay; applies popup CSS.
Popup UI
popup.html / popup.ts
Extension toolbar popup; shows last explanation; provides mode selector; links to options.
Options Page
options.html / options.ts
API key input and save; default mode selection; theme toggle (light/dark).
Chrome Storage
storage.ts (util)
Typed wrapper around chrome.storage.sync and chrome.storage.local for key/settings persistence.
AI Client
aiClient.ts (util)
Centralised API call module; handles OpenAI and Gemini endpoints; applies prompt templates; manages errors.
1.2 Message Flow
User selects text → right-clicks → clicks 'Explain with ExplainX'.
Chrome fires onClicked event to background.ts Service Worker.
Service Worker reads the selection via tabs.sendMessage() → content.ts.
Content.ts returns selectedText and injects the popup overlay with loading state.
Service Worker calls aiClient.ts → sends request to AI API.
Response returned → Service Worker sends explanation to content.ts via message.
Content.ts renders explanation inside popup. User reads, copies, or dismisses.

4. Background Service Worker (background.ts)
4.1 Responsibilities
Register context menu item on extension install.
Listen for chrome.contextMenus.onClicked events.
Send message to content script to retrieve selection + inject popup.
Call AI API via aiClient.ts with selected text and mode.
Return explanation back to content script via chrome.tabs.sendMessage().
4.2 Key Code Skeleton
// background.ts
chrome.runtime.onInstalled.addListener(() => {
  chrome.contextMenus.create({
    id: 'explainx',
    title: 'Explain with ExplainX',
    contexts: ['selection'],
  });
});
chrome.contextMenus.onClicked.addListener(async (info, tab) => {
  if (info.menuItemId !== 'explainx' || !tab?.id) return;
  const { selectedText, mode } = await chrome.tabs.sendMessage(tab.id, { type: 'GET_SELECTION' });
  const explanation = await callAI(selectedText, mode);
  chrome.tabs.sendMessage(tab.id, { type: 'SHOW_EXPLANATION', explanation });
});
5. Content Script (content.ts)
5.1 Responsibilities
Listen for GET_SELECTION message → return window.getSelection().toString() and active mode.
Listen for SHOW_EXPLANATION message → render or update the popup overlay.
Handle popup dismissal (Escape key, outside click, close button).
Render mode tab switcher; on mode switch, re-request explanation via chrome.runtime.sendMessage.

5.3 CSS Injection Strategy
The overlay is injected via content.css declared in manifest.json. All ExplainX styles use the namespace prefix #explainx- to prevent conflicts with host page styles. z-index is set to 2147483647 (maximum) to ensure the popup renders above all host page elements. Shadow DOM is not used in v1.0 for simplicity; if host CSS bleed becomes a problem in post-MVP, migrate popup to Shadow DOM.

6.2 API Call Function
// utils/aiClient.ts
export async function callAI(text: string, mode: Mode): Promise<string> {
  const { apiKey, provider } = await getStoredSettings();
  const prompt = buildPrompt(text, mode);  // from promptTemplates.ts
  const res = await fetch(API_ENDPOINTS[provider], {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${apiKey}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: MODELS[provider],
      messages: [{ role: 'user', content: prompt }],
      max_tokens: 600,
      temperature: 0.3
    })
  });
  if (!res.ok) throw new Error(`API Error: ${res.status}`);
  const data = await res.json();
  return data.choices[0].message.content;
}
6.3 Error Handling
401 Unauthorized → Display 'Invalid API key. Please check Options.'
429 Rate Limited → Display 'Too many requests. Please wait a moment.'
Network error → Display 'Connection failed. Check your internet.'
Timeout (> 10s) → Abort request, display 'Request timed out. Try again.'
Empty selection → Do not call API; show 'Please select some text first.'

8. Code Detection (utils/codeDetector.ts)
The extension auto-detects whether selected content is code to apply appropriate prompts. Detection is heuristic-based (no AST parsing in v1.0).
8.1 Detection Rules
Selection contains 2+ lines with consistent indentation (2 or 4 spaces / tab).
Selection contains keywords: function, def, class, const, let, var, return, import, for, while.
Selection contains operators: =>, ::, ->, ;
Selected text originates from a <code>, <pre>, or .highlight DOM element.
Score ≥ 2 of the above triggers code mode.
// utils/codeDetector.ts
export function isCode(text: string, element?: Element): boolean {
  let score = 0;
  const lines = text.split('\n');
  if (lines.length >= 2 && lines.some(l => l.startsWith('  ') || l.startsWith('\t'))) score++;
  const keywords = /\b(function|def|class|const|let|var|return|import|for|while)\b/;
  if (keywords.test(text)) score++;
  if (/=>|::|->|;/.test(text)) score++;
  if (element?.closest('code, pre, .highlight')) score += 2;
  return score >= 2;
}

11. Security Considerations
11.1 API Key Security
API keys are stored ONLY in chrome.storage.sync. Never in localStorage, cookies, or logs.
Keys are transmitted only to the configured AI API endpoint over HTTPS.
The options page masks the key field (type='password') after entry.
No analytics or telemetry service receives the API key.
11.2 Content Security Policy (CSP)
MV3 extensions enforce strict CSP by default. ExplainX does not use eval() or inline scripts. All scripts are bundled at build time. The extension does not load remote scripts.
11.3 Host Page Isolation
Content script runs in an isolated world — ExplainX's JavaScript cannot access the host page's JS variables or modify its DOM beyond the injected overlay.
The overlay uses CSS namespace prefix to minimize class conflicts.
Post-MVP: Migrate to Shadow DOM for full style isolation.

13. Constraints & Known Limitations
Chrome-internal pages (chrome://, chrome-extension://) cannot have content scripts injected — extension is non-functional there. This is a Chrome platform constraint.
PDF viewer pages in Chrome do not expose text selection to content scripts in the standard way. PDF support is excluded from v1.0.
Maximum input tokens sent to API is capped at ~1500 tokens (~1200 words) to control cost. Selections beyond this are truncated with a warning.
MV3 Service Workers are terminated after inactivity and re-instantiated on next event. Persistent state must be stored in chrome.storage, not memory.
Streaming AI responses require SSE handling in the Service Worker; this is a v2.0 feature.
